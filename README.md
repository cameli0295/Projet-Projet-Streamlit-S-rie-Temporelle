Projet : Projet Streamlit & Série Temporelle
Dans le cadre de l’évaluation finale du module statistiques et modèles sarimax, vous
devez réaliser en binôme (ou trinôme au maximum) une application interactive en
Python à l’aide de Streamlit.
Objectif de l’application
Votre interface Streamlit doit permettre à un utilisateur de :
1. Charger un fichier au format CSV ou Excel contenant une série temporelle depuis
son ordinateur.
2. Visualiser :
◦ le graphique de la série originale,
◦ la décomposition STL (tendance, saisonnalité, résidu),
◦ un indicateur de stationnarité (test ADF avec valeur p ou message explicatif).
3. Choisir un modèle parmi :
◦ ARIMA
◦ SARIMA
4. Paramétrer et entraîner le modèle choisi.
5. Produire une prédiction sur un horizon ( le nombre de pas de temps dans le futur
que le modèle va prédire) choisi par l’utilisateur.
6. Afficher :
◦ le graphique de la prédiction,
◦ la comparaison série observée / série prédite (superposition ou zoom sur la
période récente).
Travail à rendre
Vous devez rendre un seul fichier PDF contenant :
• Le nom du binôme ou du trinôme (maximum 3 étudiants),
• Le lien URL public donnant accès à votre application Streamlit fonctionnelle
(Streamlit Cloud, Render, HuggingFace Spaces ou équivalent).
• Délai : vendredi 23 janvier à 18h.
Aucun code n’est à envoyer, uniquement le PDF.
Règles importantes
• L’application doit être réellement fonctionnelle en ligne.
• Je n’accepterai pas les messages du type : « Ça marche sur mon PC mais pas en
ligne. »
• Toute application inaccessible sera considérée comme non rendue.
